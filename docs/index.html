Features

Everything your display needs

CASEDD ships batteries-included — getters, renderer, template engine, and a full web UI all in one lightweight daemon.

🖥️
Framebuffer output
Writes directly to /dev/fb1 via memory-mapped I/O. Zero desktop, zero X11, zero GPU overhead.
mmap/dev/fb1Waveshare 5"
🌐
Live web view
Each rendered frame is pushed over WebSocket to any browser instantly. An HTTP endpoint serves single-frame PNG exports.
WebSocketFastAPIPNG export
🎨
CSS Grid templates
Declare layouts in .casedd YAML files using CSS Grid Template Areas. A drag-and-drop editor is built into the web UI.
.casedd YAMLCSS Gridnested panels
📊
Rich widget palette
value, text, bar, gauge, histogram, sparkline, clock, image, slideshow, htop, net_ports, sysinfo, ups, apod, and more.
gaugesparklinehtopapod
🔄
Template rotation & scheduling
Cycle templates on a timer, switch by time of day, or trigger swaps from live data thresholds. Configurable per-panel.
rotationscheduletriggers
🔌
Push-based ingestion
Any external process can inject values via Unix domain socket or REST API. Perfect for custom sensors and external monitors.
Unix socketREST POSTJSON
🧠
Ollama integration
Reads active model name, VRAM usage, GPU utilization, and runtime from a local Ollama API. Built for AI rigs.

Quick Start

Up and running in minutes

No display hardware required to get started — set CASEDD_NO_FB=1 and use the browser view during development.

1
Clone & create the venv
Python 3.12 is required. Works on Ubuntu 24.04, macOS, and most Linux distros.
2
Configure via .env
Copy .env.example to .env. Set CASEDD_NO_FB=1 to skip framebuffer hardware during development.
3
Start the daemon
Open localhost:8080 for the live view, or localhost:8080/app for the full template editor.
4
Customize a template
Edit a .casedd YAML in templates/ or use the visual editor. Changes apply on the next render cycle — no restart needed.
$ git clone https://github.com/mdmoore25404/casedd.git
$ cd casedd
$ python3.12 -m venv .venv && source .venv/bin/activate
$ pip install -r requirements.txt
 
# Configure (disable framebuffer in dev)
$ cp .env.example .env
$ echo "CASEDD_NO_FB=1" >> .env
 
# Start the daemon
$ ./dev.sh start
casedd started (PID 12345)
Web viewer: http://localhost:8080
 
# Check health
$ ./dev.sh status
casedd is RUNNING (PID 12345)
